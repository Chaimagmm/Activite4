chapitre 1: Introduction

1-les techniques de l’information et de la communication (TIC):
Les techniques de l’information et de la communication regroupent l’ensemble des outils mis en place pour manipuler,>
traiter, faire circuler, transmettre, modifier et échanger (réseaux et moyens de transmission et d’échange)
La naissance des TIC est due à la convergence de l’informatique, des télécommunications, des électroniques et de l’audiovisuel
2-catégories et objectifs :
Les catégories de TIC sont : la téléinformatique, Télécommunication, l’électronique.
Les objectifs recherchés sont : la connectivité, la modularité, une
implantation simple, une utilisation facile, la fiabilité.
3- les composants de TIC :
Il composé en deux partie matérielle et logicielle.
4-Caractéristiques:
La rapidité, la décentralisation du pouvoir, la rationalité, la mondialisation, la répartition et l'intensité de l'information
donnent un avantage pour l'entreprise.
5- Objectif de communication :
Les objectifs de communication doivent être définis selon trois dimensions: cognitif, affectif et comportemental. L'objectif cognitif
est de prendre de conscience l'attention, la notoriété et l'information, l'objectif affectif est de modifier l'image de produit et l'objectif
comportemental est de modifier les comportements.
Système de communication :
L'émetteur transforme le message en signaux, le récepteur reçoit les signaux et reconstruit le message. Les vecteurs sont utilisés pour transmettre
le message. Les fonctions sont codage, décodage, réponse, feedback et bruit.
Les types de communication :
La communication est un outil indispensable pour tous organismes, et nous distinguons deux types : la communication institutionnelle (corporate)
et la communication commerciale ou de produit.
Les schémas de communication:
La communication interpersonnelle est utilisée pour transmettre un message à un destinataire. La communication de groupe est utilisée pour transmettre
un message à plusieurs destinataires. La communication de masse est utilisée pour publiciser une entreprise.
Media et Multimedia :
Média et Multimédia sont deux techniques et produits qui permettent l'utilisation simultanée et interactive de
différentes modes de représentation de l'information.

Chapitre 2: Les réseaux informatiques

1/ Réseau (informatique) Network:
Un réseau est un ensemble d'ordinateurs autonomes connectés dans un domaine géographique, échangeant
des informations numériques selon des règles définies, et permettant la communication de données entre applications. Deux ordinateurs
connectés forment déjà un réseau.
La mise en réseau (Networking) : implique la mise en œuvre d'outils et de tâches pour relier les ordinateurs et permettre le partage de ressources.
2/ Les caractéristiques de base d’un réseau sont :

  • Le débit exprimé en bits/s (ou bps) qui mesure une quantité de données numériques La topologie d'un réseau inclut sa structure physique
    (interconnexions d'équipements) et logique (mode de communication entre équipements).

  • Le débit exprimé en bits/s (ou bps) qui mesure une quantité de données numériques (bits) transmises par seconde (s).

  • La distance maximale (ou portée) qui dépend de la technologie mise en œuvre.

  • Le nombre de nœuds maximum que l’on peut interconnecter.

    • 3-Quelques terminologies en réseau:
    Station de travail : une machine capable d'envoyer des données vers les réseaux.
  • Nœud : Station de travail, imprimante, serveur ou entité adressable par numéro unique.

  • Serveur : Dépositaire d'une fonction

    • spécifique (base de données, calcul, fichier, mail, etc...).
    Paquet: Plus petite unité d'information envoyée sur réseau. Contient adresse de l'émetteur, du récepteur et les données à transmettre.
    Réseaux Homogènes: Tous les ordinateurs sont du même constructeur. Ex: AppleTalk (système utilisé par ordinateurs Apple Macintosh).
    Réseaux Hétérogènes : Les ordinateurs reliés au réseau sont de constructeurs divers: Ethernet
    4/ Éléments d'un reseau:
  • Les ordinateurs équipés d'une carte de communication

  • Les logiciels navigateur, client de messagerie,

  • Les supports du : LAN : câbles paires cuivre torsadées , WIFI , ligne
    Téléphonique , ADSL

    • Les équipements d'interconnexion du LAN : répéteur (transceiver), concentrateur (hub),
    commutateur (switch) WAN : routeur
    5/ Classification des réseaux :
    Les réseaux peuvent être classifiés selon différents critères :
    5.1. Par portée l’envergure (La distance) :
    Par portée l’envergure (la distance) : LAN, MAN et WAN
  • LAN (Local Area Network = réseau local d'entreprise) :Réseau local d'entreprise, couvrant quelques centaines de mètres à quelques kilomètres.
    Connecte jusqu'à environ une centaine de machines avec des vitesses de transmission de 10 à 100 Mb/s.

  • MAN : réseau métropolitain ou urbain . La distance de connexion typique dans un MAN est de 10 à 25 km.
    Le câble co-axial est le support physique le plus utilisé dans ce type de réseau.

  • WAN (Wide Area Network) : Réseau étendu à grande échelle( d'un pays ou de la planète).
    Géré par des opérateurs publics ou privés.
    • Autres types de réseaux : PAN (Personal Area Network), WPAN et WLAN et SAN .
    5.2. L'ouverture et l'utilisation des réseaux :
  • : Réseau interne d'une entité organisationnelle ou d'une entreprise.

  • Extranet : Réseau externe d'une entité organisationnelle, ouvert vers l'extérieur.

  • Internet : Réseau public interconnectant des réseaux nationaux et internationaux à l'échelle de la planète.

    • 5.3. Les supports de connexion :
    filaires et non filaires :
  • Réseaux filaires : utilisation de différents types de câbles, tels que le câble coaxial.

  • Réseaux sans fil : utilisation d'autres moyens de transmission, tels que les ondes radio.

    • 5.4. Le type d’organisation :
  • Réseaux Peer to Peer : Chaque ordinateur agit comme serveur et client, permettant le partage de ressources. Moins sécurisés que les autres types de réseaux.

  • Réseaux client-serveur : La plupart des ordinateurs sont des clients et certains sont dédiés à des tâches spécifiques en tant que serveurs. L'administration du
    réseau est effectuée par un administrateur.

    • 6/ Le type de topologie :
  • Réseaux en bus : Les machines sont reliées en série par un câble coaxial, appelé bus. . Débit limité à 10 Mbits/s.

  • Réseaux en étoile : Toutes les stations sont connectées à un commutateur central (switch) ou (hub). Les données sont envoyées vers le commutateur qui
    les renvoie à tous les autres ports (hub) ou uniquement au destinataire (switch).

  • Réseaux en anneau : Les ordinateurs sont disposés en boucle. . Le jeton détermine quelle
    station peut émettre, il est transféré à tour de rôle vers station suivante.

  • Topologie maillée : Utilisée dans les réseaux MAN et WAN. Les réseaux maillés utilisent plusieurs chemins de transfert entre les nœuds, garantissant la stabilité en cas de
    panne d'un nœud. Combinaison d'une architecture en étoile avec des câblages reliant différents nœuds.

    • Chapitre 3: Le modèle OSI

    1/les composantes matériel d’un réseau informatique :
    Les cartes réseau :
    Également appelées cartes d’interface réseau (NIC) sont des composants matériels qui permettent aux ordinateurs de se connecter à différents réseaux, elles permettent
    la transmission et la réception de données à travers des protocoles de communication.
    SWITCH :
    Est un composant matériel utilise pour connecter plusieurs périphériques sur un réseau local (LAN). Il permet aussi d’envoyer des données uniquement a l’appareil
    destinataire prévu et cela a l’aide d’adresses MAC et d’une table de correspondance, il peut également offrir des fonctionnalités plus avancées.
    HUB :
    Le HUB est un composant matériel qui fonctionne comme un répéteur mais qui n’a pas la même fonctionnalité qu’un SWITCH, il recoit des données d’un appareil et les transmet
    à tous les autres appareils qui lui sont connectes.
    2/ Le Modèle OSI (Open Systems Interconnection)
    est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes comme suit :

  • La couche physique : elle est responsable de la transmission des données sur le support physique.
  • La couche liaison de données : elle transmit les données entre les appareils pour même support physique.
  • La couche réseau : elle est responsable du routage des données.
  • La couche transport : elle est responsable de l'acheminement des données.
  • La couche session : elle est responsable de la gestion des sessions.
  • La couche présentation : elle est responsable de la conversion des données dans un format.
  • La couche application : cette couche est responsable de la communication entre les applications.
    • Le modèle OSI reste utile pour comprendre les principes fondamentaux de la communication de réseau.
    3/LE MODELE TCP/IP :
    est un modèle de référence pour les communications réseau utilise dans les réseaux informatiques modernes. Il diffère du modèle OSI en divisant
    les communications en 4 couches principales :
  • LA COUCHE LIAISON DE DONNEES gère la transmission physique des données.
  • LA COUCHE RESEAU achemine les données sur le réseau.
  • LA COUCHE TRANSPORT transmet les donnes entre les appareils.
  • LA COUCHE APPLICATION gère la communication entre les applications réseau.
    • 4/ L’adressage :
    L’adressage les communications dans un réseau obéit à des règles : Adressage, Architecture et Protocoles
    Une adresse IP (Internet Protocol) est le numéro qui identifie chaque ordinateur dans les réseaux qu’il appartient. Il y’a deux mode d’adressage I:
    1. ADRESSE IPV4: Chaque paquet contient : adresse source sur (32 bits) et une adresse destinataire sur (32bits)
    Donc IPV 4 == 4 octets ==32 bits
    2. ADRESSE IPV6 : Les adresses sont codées sur 128 bits.

    5/ADDRESSE IPV4 est divisée en 2 parties :
    L’adresse réseau et l’adresse hôte
    Les réseaux sont identifiés par le fait que les bits de la partie réseau sont identiques, et le nombre de bits dans la partie hôte détermine
    le nombre de machines pouvant se trouver sur ce réseau.
    Les 32 bits sont divisés en 4 octets et convertis-en décimal.
    Il existe 4 types d’adressage IPV4 :
  • L’adresse réseau : tous les bits de la partie hôtes = 0.

  • L’adresse de diffusion : tous les bits de la partie hôtes = 1.

  • L’adresse hôtes : les valeurs situées entre l’adresse réseau et l’adresse de diffusion sont attribuées aux machines situées sur ce réseau.
  • Le masque de réseau et de sous-réseaux : tous les bits de la partie réseau = 1.

    • Chapitre 4: L'Internet et le Web

    Introduction :
    Le World Wide Web est un système hypertexte public sur Internet qui permet de consulter des pages accessibles sur des sites. Les pages web sont
    reliées entre elles par des hyperliens, ce qui donne l'image d'une toile d'araignée.
    Le principe du World Wide Web est de permettre l'accès à des documents textuels situés sur des machines accessibles par Internet et reliés entre
    eux par un mécanisme de lien hypertexte.
    Une URL contient l'adresse du serveur ainsi que l'adresse de la page sur le serveur
    Aspects technique de web :

  • URL: est utilisée pour identifier de manière unique une ressource sur le Web. De plus, le protocole DNS qui est utilisé pour traduire les noms de
    domaine en adresses IP afin de localiser les serveurs hébergeant les ressources.

  • HTTP: utilisé pour transférer les données sur le Web. Il utilise un modèle client-serveur et il est sans état.

  • HTML :utilisé pour créer des pages Web, y compris les liens hypertextes qui sont des éléments clés du Web. HTML est un langage de programmation déclarative,
    ce qui signifie que les instructions sont fournies pour décrire le résultat souhaité plutôt que les étapes pour y parvenir.

    • Une page web et un site web :
    Une page web est un document multimédia il est décrite élément par élément à l'aide de langages de description tels que HTML. Elle est stockée dans
    un fichier et est interprétée et affichée par un navigateur, localisée sur Internet à l'aide d'une adresse URL, et permet d'accéder à d'autres pages en suivant des liens hypertextes.
    Un site web, est un ensemble de pages web regroupées autour d'une thématique commune et reliées entre elles par des liens hypertextes. Il émane d'une même entité et est accessible à partir
    d'une page d'accueil commune et d'une URL de base commune. Les pages sont généralement localisées sur un même serveur et structurées sous forme d'arborescence de dossiers et de fichiers,
    permettant de définir des chemins relatifs.
    Creation d’une page web html:
    HTML est un langage de description de pages web qui utilise des balises pour structurer le contenu d'un document. Un document HTML est un fichier texte construit suivant une syntaxe précise,
    qui est interprété et visualisé dans un navigateur.
    Les principes de base d'HTML comprennent l'utilisation de balises, qui permettent de délimiter le début et la fin du document, l'entête du document, le titre du document et le corps du document.
    Le texte explique également comment utiliser les balises pour créer des titres de niveau 1 à 6 et des paragraphes.

    Chapitre 5: Système de Gestion de Version

    1. Qu’est-ce que Git ?
      2. Git est un logiciel de versioning (logiciel de gestion de version) qui conserve les modifications d’un projet afin de revenir à l’ancienne version en cas de problème.
      Aujourd’hui Les logiciels de versioning facilitent la gestion de projets et permettent de travailler en équipe et parmi eux Git est le plus indispensable pour tout développeur.
    2. A quoi sert un système de gestion de version ?

      3. les logiciels de versioning effectuent plusieurs opérations :
      - Synchroniser son dossier local avec le serveur.
      - Connexion aux autres projets.
      - Mise à jour des différences automatisée, etc.
      Git permet de :
      - coordonner le travail entre plusieurs personnes en conservant un
      historique des changements des fichiers.
      - Coexistence de différentes versions d’un même fichier. plusieurs développeurs
      ont accès à l’historique des modifications.
    3. Les deux modèles des logiciels de gestion de version :

      4. Un modèle centralisé : le code source est lié au projet : la source est hébergée sur un serveur distant central dont les utilisateurs se connectent pour travailler sur ce code.
      Un modèle décentralisé (distribué) : le code source du projet est sur un serveur distant que chaque utilisateur héberge sur sa machine.
      Le modèle distribué présente différents avantages :
      • Simplicité / flexibilité du travail on peut travailler en ligne sur sa propre machine.
      • Sécurité : on peut utiliser la copie du projet d’un utilisateur comme back- up en cas de corruption du serveur central.
    4. Qu’est-ce que GitHub ?

      5. GitHub est un service en ligne d’hébergement de dépôts Git (dont la plupart sont public) qui fait office de serveur central pour ces dépôts.
    5. Utiliser Git :
      6.  ligne de commande, console et interface Graphique
      Utiliser la console et un langage en ligne de commande plutôt qu’une interface graphique car :
      • Utiliser la ligne de commande a accès à toutes les commandes Git ;
      • Toutes les personnes auront accès aux mêmes commandes.
      • On peut utiliser n’importe quelle autre interface graphique en métrisant la ligne de commande.
      En Windows, pour ouvrir la ligne de commande :
      1. Raccourcie : (Windows) + (r) + cmd
      2. Dans la barre de recherche, rechercher cmd
    6. Installation de Git:

      7. Il faut télécharger la dernière version sur le site officiel Cliquez ici pour télécharger Git , ouvrir le fichier et suivre les instructions à l’écran.
    7. Paramétrage de Git:

      8. Il faut :
      - Utiliser la commande « git config » pour voir et modifier les variables de configuration contrôlant tous les aspects de l’apparence et du comportement de Git.
      - passer une option --global à notre commande
      - renseigner un nom et une adresse email. (tapez git config --global user.name "votre nom" et git config --global user.email ‘’votrenom‘’@gmail.com)
    8. Démarrer un dépôt Git:

      9. Il existe deux façons de créer un dépôt Git :
      • On peut importer un répertoire déjà existant dans Git ;
      • On peut cloner un dépôt Git déjà existant.
    9. La gestion des informations selon Git:

      10. Git pense les données à la manière d’un flux d’instantanés ou “snapshots”.
      Chaque instantané est stocké dans une base de données locale pour des opérations rapides et un travail agréable.
    10. Les états des fichiers:

      11. un fichier peut avoir deux grands états dans Git :
      l’état “suivi’’ : le fichier appartient au dernier instantané capturé par Git.
      l’état “non suivi” : le fichier n’appartenait pas au dernier instantané et non indexé.
    11. Les zones de travail:

      12. Les zones de travail sont liées au fichier en fonction de leur état
      Tout projet Git est composé de :
      - répertoire de travail (working tree) : correspond à une extraction unique (“checkout”) les fichiers de la base sont placés sur le disque
      - zone d’index (staging area) : correspond à un simple fichier qui stocke les informations concernant ce qui fera partie du prochain instantané.
      - répertoire Git (repository) : Est la partie principale de git ou on stocke les méta-données et la base de données du projet.
    12. Créer un dépôt Git à partir d’un répertoire existant:

      13. Si on a des projets en cours stockés localement sur notre machine ,ou sur serveur distant et on veut implémenter un système de gestion de version,
      on importe l’ensemble des ressources d’un projet dans Git.
      Note : Dans cmd les commandes :
      • cd : se positionner à un emplacement dans le PC.
      • dir : afficher la liste des dossiers .
      - Pour initialiser un dépôt Git, on utilise la commande git init en créant un répertoire, alors on reçoit un message que le dépôt a été initialisé et qu’il est vide.
      - On utilise la commande git status pour déterminer l’état des fichiers de notre répertoire.
      - Pour indexer des fichiers, on utilise la commande git add. On peut lui passer un nom de fichier pour indexer le fichier en question.
      - Pour valider les fichiers et les ajouter en base, on utilise la commande git commit.
      - Les fileglobs utilisent les extensions de chemin de fichier.
      - On écrit git add ‘’nom du fichier’’ pour demander à Git d’ajouter à l’index tous les fichiers du projet qui possèdent une extension .
    13. Cloner un dépôt Git:

      14. La deuxième façon de démarrer un dépôt Git est de cloner localement, on va utiliser la commande Git clone, créer un dépôt et transmettre les fichiers.
    14. Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git:

      15. Dans un projet on peut ajouter, modifier, supprimer des fichiers donc on demande a git de conserver un historique des versions au quel on peut accéder.
      on utilise les commandes git add et git pour une modification ou un ajout d’un fichier, son commit se basera sur l’état au moment du git add.
      Le git commit et Les modifications effectués après git add ne seront pas enregistrées dans le dépôt, le fichier reste à l’état du dernier git add.
      Pour enregistrer un fichier il faut effectuer un git add avant un git commit, et pour mettre les fichiers modifiés et sous suivi dans la zone
      d’index et les valider on utilise un git commit avec l’option -a.
    15. Consulter l’historique des modifications Git:
      16.
      pour consulter l’historique des modifications Git on utilise la commande git log qui affiche les commits réalisés du plus récent au plus ancien.
      chaque commit est affiché avec sa somme de contrôle SHA-1, le nom et l’e-mail de l’auteur, la date et le message du commit.
    16. Annuler des modifications apportées à un fichier :
      17.
      On utilise un logiciel de gestion de vision pour pouvoir revenir à l’état antérieur enregistré d’un projet, ou ‘’roll back’’.
      Si les modifications n’apportent pas de les bons comportement et on veut revenir au dernier état du fichier on utilise git checkout – nom du fichier, ou git restore.
    17. Qu’est ce qu’une branche ?

    Dans Git, une branche est un pointeur vers le dernier commit.
    Un commit est un objet qui contient un pointeur vers l’instantané du contenu indexé ainsi que des pointeurs vers les commits précédents.
    Créer une nouvelle branche dans Git crée simplement un nouveau pointeur plutôt que de recopier l’intégralité du répertoire de travail.
